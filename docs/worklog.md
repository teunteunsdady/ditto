# 개발 작업일지

## 2026-04-12

### 목표
- Next.js + Supabase 기반의 무료 운영 가능한 초기 프로젝트 세팅
- 향후 인코칭 레퍼런스 사이트 구조를 반영할 수 있는 기반 만들기

### 진행 내용
- Next.js 14(App Router, TypeScript, Tailwind, ESLint) 프로젝트 생성
- Supabase SDK 설치: `@supabase/supabase-js`, `@supabase/ssr`
- Supabase 클라이언트 유틸 추가
  - `src/lib/supabase/client.ts`
  - `src/lib/supabase/server.ts`
  - `src/lib/supabase/middleware.ts`
  - `middleware.ts`
- 환경변수 템플릿 추가: `.env.example`
- 초기 메인 페이지를 프로젝트 안내형 레이아웃으로 교체
- Supabase 세팅 문서 추가: `docs/supabase-setup.md`

### 의사결정
- **백엔드 전략**: 별도 Express/Nest 서버 없이 Supabase를 메인 백엔드로 사용
- **서버 로직 위치**: Next.js Route Handler/Server Action + Supabase 조합
- **비용 전략**: Vercel + Supabase Free 플랜 범위 내 시작

### 다음 작업
- 홈 UI를 레퍼런스 구조(코칭/진단/교육/인사이트) 중심으로 구체화
- 문의 폼 구현 + DB 저장 API(Route Handler) 연결
- 인사이트 목록/상세 페이지 구현 및 CMS 운영 방식 결정
